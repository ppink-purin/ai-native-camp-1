## 🥇 추천 #1 (점수: 9.8/10)

### Claude Code 팀이 도구를 3번 갈아엎고 배운 Tool 설계 원칙 4가지

**작성자:** Jeongmin Lee
**직책:** Control opinions with AI @ Smoretalk
**반응:** 👍 34+ | 💬 2 | 🔄 6
**게시 시간:** 2시간 전 (수정됨)

**한 줄 요약:**
Anthropic Claude Code 팀의 1년간 Tool 설계 시행착오와 4가지 핵심 원칙

**추천 이유:**
AI/ML 분야에서 가장 실용적이고 깊이 있는 게시물입니다. Anthropic 개발자 Thariq의 실제 경험을 바탕으로, AI Agent 도구 설계의 핵심 원칙을 정리했습니다. Claude Code를 사용하는 개발자라면 반드시 읽어야 할 글입니다.

**핵심 인사이트 4가지:**

1. **도구 하나에 역할 하나** - 계획+질문을 합치면 AI가 혼란. AskUserQuestion을 분리해서 해결
2. **도구 유통기한(Tool Decay)** - 모델이 업그레이드되면 기존 도구가 병목이 됨. TodoWrite → Task Tool 교체
3. **단계적 맥락 발견(Progressive Disclosure)** - RAG보다 AI가 직접 Grep으로 검색하게 하는 게 나음
4. **구조 변경으로 해결** - 도구를 더하지 않고 서브에이전트 구조로 바꿔서 기능 확장

**원문 발췌:**
> "도구를 더하고 빼고 다시 설계하는 이 과정에 정답 공식은 없습니다. 모델이 바뀌면 도구도 바뀌어야 하고 어제 맞았던 구조가 내일은 병목이 될 수 있어요. 결국 에이전트를 잘 만드는 사람은 AI 입장에서 볼 줄 아는 사람입니다."

**참고 링크:** Thariq x.com post (원문에 링크 포함)

## 🥈 추천 #2 (점수: 8.5/10)

### AI에 의해 빠르게 대체되는 인재들의 특징

**작성자:** Seunghoon Lee
**직책:** Co Founder at Ringle (1:1 Online English Education Service)
**반응:** 👍 164+ | 💬 4 | 🔄 10
**게시 시간:** 8시간 전

**한 줄 요약:**
AI 시대에 대체되는 사람과 살아남는 사람의 차이는 "태도"

**추천 이유:**
AI가 실제로 업무 현장에서 어떤 역할을 대체하고 있는지에 대한 현실적인 관찰입니다. 기술적 내용은 아니지만, AI 시대의 인재상과 협업 방식 변화를 이해하는 데 도움이 됩니다. 반응 수가 164개로 높은 공감을 받고 있습니다.

**핵심 메시지:**
- AI가 대체하는 것: 일 부탁하기 힘들고, 부탁한 일 대신 하고 싶은 일만 하고, 대충 해오는 사람
- AI가 대체 못하는 것: 합이 잘 맞고 시너지가 나는 협업 능력
- 결론: "AI 시대는 태도가 더 중요해지는 시대"

**원문 발췌:**
> "AI 시대는 AI가 대체할 수 없는 태도를 가진 사람들이 더 빛나고, AI가 대체해줬으면 하는 사람들은 설 곳을 잃는 방향으로 흘러갈 것이다."
