# My Content Digest Skill

가져온 콘텐츠(트윗, YouTube, 아티클 등)를 Quiz-First 방식으로 소화하는 스킬입니다.

## 핵심 원칙: Quiz-First 학습법

**요약을 먼저 보여주지 않습니다!**

연구에 따르면 학습 전 테스트(Pretesting Effect)가 기억력을 9-12% 향상시킵니다:
- **Pretesting Effect**: 학습 전 테스트가 기억력 향상 (Richland et al.)
- **Information Gap Theory**: 틀린 문제가 호기심을 만들고, 호기심이 도파민 → 기억 강화 (Loewenstein, 1994)

```
❌ 일반적: 요약 읽기 → 퀴즈
✅ Quiz-First: 퀴즈 먼저 → 틀린 부분 확인 → 선택적 읽기
```

**간단히 말해: 틀려야 배운다.**

---

## Step 1: Pre-Quiz (콘텐츠 보기 전 퀴즈)

### 목적

사용자가 요약을 보기 전에 퀴즈를 풀면서:
- 오답으로 호기심을 자극
- "왜 틀렸지?"라는 Information Gap 생성
- 이 상태에서 콘텐츠를 읽으면 기억력 향상

### 출제 방법

1. **콘텐츠에서 핵심 주제 3개 추출**
   - 핵심 메시지
   - 저자의 주요 주장
   - 주요 원칙/개념

2. **3문제 출제**
   - 각 문제는 4지선다 형식
   - AskUserQuestion 도구 사용
   - label: 선택지 (간결하게)
   - description: 해당 선택지의 의미 설명

3. **출제 원칙**
   - ✅ 핵심 메시지, 주요 원칙, 저자 핵심 주장
   - ❌ 날짜, 통계 수치, 지엽적 세부사항

### AskUserQuestion 예시

```json
AskUserQuestion({
  "questions": [
    {
      "question": "이 콘텐츠의 핵심 메시지는 무엇인가요?",
      "header": "Pre-Quiz 1/3",
      "multiSelect": false,
      "options": [
        {"label": "선택지 1", "description": "설명 1"},
        {"label": "선택지 2", "description": "설명 2"},
        {"label": "선택지 3", "description": "설명 3"},
        {"label": "선택지 4", "description": "설명 4"}
      ]
    }
  ]
})
```

### 즉시 피드백

각 문제 답변 후:
- ✅ **정답**: "정확합니다! [간단한 추가 설명]"
- ❌ **오답**: "아쉽습니다. 정답은 [X]번입니다. [이유 설명] 이 부분을 콘텐츠에서 확인해보세요."

Pre-Quiz 3문제를 모두 출제한 후 → Step 2로 이동

## Step 2: 선택적 콘텐츠 제공

Pre-Quiz 결과에 따라 사용자가 원하는 방식으로 콘텐츠를 제공합니다.

### 선택지 (AskUserQuestion)

```json
AskUserQuestion({
  "questions": [
    {
      "question": "어떤 방식으로 콘텐츠를 확인하시겠어요?",
      "header": "콘텐츠 제공",
      "multiSelect": false,
      "options": [
        {
          "label": "틀린 문제 관련 섹션만",
          "description": "오답 부분의 답을 집중적으로 확인"
        },
        {
          "label": "핵심 인사이트 3개",
          "description": "가장 중요한 포인트만 빠르게 파악"
        },
        {
          "label": "전체 요약 + 인사이트",
          "description": "콘텐츠 전체를 종합적으로 분석"
        },
        {
          "label": "바로 본 퀴즈로",
          "description": "요약 없이 9문제 도전 (고수용)"
        }
      ]
    }
  ]
})
```

### 선택에 따른 제공 내용

#### 1. "틀린 문제 관련 섹션만"

Pre-Quiz에서 틀린 문제와 관련된 부분만 추출하여 제공:

```markdown
## 틀린 문제 해설

### 문제 [번호]: [문제 내용]
- **정답**: [선택지]
- **이유**: [상세 설명]
- **콘텐츠 원문**: "[관련 부분 인용]"
```

#### 2. "핵심 인사이트 3개"

콘텐츠의 가장 중요한 3가지 포인트:

```markdown
## 핵심 인사이트

### 1. 핵심 메시지
[이 콘텐츠가 정말 말하고 싶은 것]

### 2. 시사점
[업계/트렌드에서의 의미, 변화 흐름]

### 3. 적용점
[나(독자)에게 어떤 의미인지, 실제 활용 방법]
```

#### 3. "전체 요약 + 인사이트"

fetch 스킬의 번역 파이프라인 결과를 모두 제공:
- 요약 (3-5문장)
- 핵심 인사이트 3개
- 전체 번역/원문

#### 4. "바로 본 퀴즈로"

요약 없이 Step 3으로 바로 이동

---

## Step 3: 본 퀴즈 (9문제)

3단계 난이도 × 3문제 = 총 9문제를 출제합니다.

### 난이도별 출제 원칙

| 난이도 | 출제 범위 | 예시 |
|--------|----------|------|
| **기본** (3문제) | 핵심 메시지, 주요 개념 | "이 콘텐츠의 주요 주장은?", "핵심 개념은?" |
| **중급** (3문제) | 개념 간 관계, 근거 연결 | "A와 B의 관계는?", "주장의 근거는?" |
| **심화** (3문제) | 실제 적용, 핵심 원리 확장 | "이 원리를 X 상황에 적용하면?", "Y와 비교하면?" |

### 출제 금지 사항

- ❌ 지엽적 세부사항
- ❌ 날짜, 통계 수치
- ❌ 콘텐츠에 명시되지 않은 추론

### 단계별 진행

각 난이도별로 AskUserQuestion을 사용하여 3문제를 한 번에 출제합니다.

**기본 단계 (3문제)**

```json
AskUserQuestion({
  "questions": [
    {
      "question": "[기본 문제 1]",
      "header": "기본 1/3",
      "multiSelect": false,
      "options": [...]
    },
    {
      "question": "[기본 문제 2]",
      "header": "기본 2/3",
      "multiSelect": false,
      "options": [...]
    },
    {
      "question": "[기본 문제 3]",
      "header": "기본 3/3",
      "multiSelect": false,
      "options": [...]
    }
  ]
})
```

**답변 후 즉시 정답/해설 제공**

각 문제에 대해:
- 정답 여부
- 정답 번호와 선택지
- 상세 해설 (왜 그런지, 콘텐츠의 어느 부분에서 나왔는지)

**중급 단계 (3문제)** → 같은 방식

**심화 단계 (3문제)** → 같은 방식

### 본 퀴즈 결과 요약

9문제 완료 후:

```markdown
## 본 퀴즈 결과

- **정답**: X / 9
- **정답률**: X%

### 난이도별 성적
- 기본: X / 3
- 중급: X / 3
- 심화: X / 3

### 강점
[잘한 부분]

### 보완점
[틀린 문제 패턴 분석]
```

## Step 4: 후속 선택

본 퀴즈 완료 후 사용자에게 다음 선택지를 제공합니다.

```json
AskUserQuestion({
  "questions": [
    {
      "question": "다음에 무엇을 하시겠어요?",
      "header": "후속 선택",
      "multiSelect": false,
      "options": [
        {
          "label": "틀린 문제 다시 풀기",
          "description": "오답 문제만 모아서 재도전"
        },
        {
          "label": "다른 난이도로 재퀴즈",
          "description": "같은 콘텐츠, 다른 각도의 문제"
        },
        {
          "label": "관련 콘텐츠 추천",
          "description": "이 주제와 연관된 다른 콘텐츠 제안"
        },
        {
          "label": "종료",
          "description": "학습 완료"
        }
      ]
    }
  ]
})
```

### 선택에 따른 후속 작업

#### 1. "틀린 문제 다시 풀기"
- 본 퀴즈에서 틀린 문제만 추출
- 선택지 순서를 섞어서 재출제
- 2회차에도 틀리면 상세 해설 제공

#### 2. "다른 난이도로 재퀴즈"
- 같은 콘텐츠에서 새로운 각도의 문제 생성
- 예: 기본을 다 맞췄으면 중급/심화 중심으로
- 심화를 틀렸으면 개념 복습 문제 추가

#### 3. "관련 콘텐츠 추천"
- 이 콘텐츠의 주제와 관련된 다른 콘텐츠 제안
- 예: 같은 저자, 같은 주제, 반대 관점

#### 4. "종료"
- 학습 세션 종료
- 최종 요약 메시지:

```markdown
## 학습 완료! 🎉

오늘 학습한 내용:
- Pre-Quiz: X / 3
- 본 퀴즈: X / 9
- 총 정답률: X%

Quiz-First 방식으로 [콘텐츠 제목]을 소화했습니다.
틀린 문제에서 호기심이 생기고, 그 호기심이 기억을 강화했습니다.

수고하셨습니다!
```

## 실행 흐름 요약

```
┌─────────────────────────────────────────┐
│  1. 콘텐츠 받기 (fetch 스킬 결과물)       │
├─────────────────────────────────────────┤
│  2. Pre-Quiz (3문제)                     │
│     ↓ 요약 보기 전에 퀴즈부터!            │
├─────────────────────────────────────────┤
│  3. 선택적 콘텐츠 제공                    │
│     - 틀린 부분만                         │
│     - 핵심 인사이트                       │
│     - 전체 요약                           │
│     - 바로 본 퀴즈로                      │
├─────────────────────────────────────────┤
│  4. 본 퀴즈 (9문제)                      │
│     - 기본 3문제                          │
│     - 중급 3문제                          │
│     - 심화 3문제                          │
├─────────────────────────────────────────┤
│  5. 결과 + 후속 선택                      │
│     - 재퀴즈 / 추천 / 종료                │
└─────────────────────────────────────────┘
```

## 사용 예시

### fetch-tweet과 연결

```
1. "이 트윗 번역해줘 [URL]" → fetch-tweet 실행
2. 번역 완료 후 → "이 트윗 내용으로 퀴즈 내줘"
3. my-content-digest 자동 실행 → Pre-Quiz부터 시작
```

### fetch-youtube와 연결

```
1. "이 YouTube 번역해줘 [URL]" → fetch-youtube 실행
2. 번역 완료 후 → "영상 내용으로 학습하기"
3. my-content-digest 자동 실행 → Quiz-First 학습
```

## 핵심 원칙 (반복 강조)

1. **요약을 먼저 보여주지 않는다** — Pre-Quiz가 먼저
2. **틀려야 배운다** — 오답이 호기심을 만든다
3. **선택적 깊이 탐색** — 사용자가 원하는 만큼만
4. **핵심만 출제** — 날짜/수치 같은 지엽적 정보는 제외
5. **즉시 피드백** — 각 문제 답변 후 바로 해설
